# 执行清单

> 每完成一项，我会把未勾选状态改成已勾选状态。每一项都附有你可以怎么检查的方式，方便你判断这个系统是否值得信任。

## 1. 项目骨架与中文文档

- [x] 初始化 git、Python 项目结构和中文 README。

你可以这样检查：

```bash
git status --short
find . -maxdepth 3 -type f | sort
cat README.md
```

可信标准：

- 项目有 `pyproject.toml`。
- 代码放在 `src/sourcing_agent/`。
- 测试放在 `tests/`。
- README 用中文解释核心工作流和运行方式。

## 2. 数据模型、配置与存储

- [x] 实现 `SourceItem`、`ProcessedSignal`、`ScoreResult`、`OutreachDraft` 等基础模型。
- [x] 实现配置读取和 SQLite 存储。

你可以这样检查：

```bash
PYTHONPATH=src python3 -m unittest tests.test_models tests.test_config tests.test_storage
```

可信标准：

- 不同来源只统一外壳，不硬塞成同一种公司结构。
- 数据能写入 SQLite 并读出。
- 配置文件能被读取，且阈值可改。

- [x] 实现 YC/网页、GitHub、arXiv、X/HN 的采集解析基础能力。
- [x] 实现 artifact type 分流：`company / repo / paper / post`。

你可以这样检查：

```bash
PYTHONPATH=src python3 -m unittest tests.test_sources tests.test_routing
```

可信标准：

- GitHub 项目进入 `repo`。
- arXiv 论文进入 `paper`。
- YC/VC/官网内容进入 `company`。
- X/HN 讨论进入 `post`。
