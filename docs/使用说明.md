# 使用说明

## 这个系统现在能做什么

当前版本是一个可运行 MVP，完成了这条链路：

```text
样例/公开来源信息
  -> SourceItem 外壳
  -> artifact_type 分流
  -> company / repo / paper / post processor
  -> scoring rubric
  -> digest
  -> 建联草稿
  -> 人工审核队列
```

注意：系统不会自动发送邮件，只会生成草稿。

## 为什么不把所有来源硬转成统一结构

因为不同来源的信息本质不同：

- YC / VC / 官网更像创业公司线索。
- GitHub 更像技术项目和开发者势能线索。
- arXiv 更像研究和商业化潜力线索。
- X / HN 更像早期热度和 launch 信号。

所以代码只统一最外层：

```text
source
source_kind
artifact_type
title
url
raw_text
published_at
metadata
```

进入 processor 后再按类型处理。

## 主要文件

```text
src/sourcing_agent/models.py          # SourceItem / ProcessedSignal / ScoreResult / OutreachDraft
src/sourcing_agent/sources/           # GitHub / arXiv / X / HN / webpage 解析
src/sourcing_agent/processors/        # company / repo / paper / post 处理逻辑
src/sourcing_agent/scoring/score.py   # scoring rubric、cap、penalty
src/sourcing_agent/outputs/           # memo、digest、中英文建联模板
src/sourcing_agent/workflow.py        # 每日运行主流程
src/sourcing_agent/cli.py             # 命令行入口
src/sourcing_agent/review/api.py      # 人工审核队列
src/sourcing_agent/scheduler.py       # 给 cron / Codex automation 复用的入口说明
```

## 如何运行

先跑测试：

```bash
PYTHONPATH=src python3 -m unittest discover -s tests
```

跑一轮确定性 dry-run：

```bash
PYTHONPATH=src python3 -m sourcing_agent.cli run-daily --dry-run
```

查看 digest：

```bash
PYTHONPATH=src python3 -m sourcing_agent.cli show-digest
```

查看待人工审核的建联草稿：

```bash
PYTHONPATH=src python3 -m sourcing_agent.cli review-queue
```

## 如何判断结果是否可信

1. 看 `docs/执行清单.md`。
2. 跑每个阶段对应的测试命令。
3. 检查 digest 中每条线索是否包含：
   - 类型
   - 来源
   - URL
   - raw score
   - outreach score
   - 摘要
4. 检查建联草稿是否只出现在有明确邮箱联系方式的项目上。

## Scoring 逻辑

系统会先按类型打分：

- company：主题匹配、创始人信号、产品清晰度、牵引力、时机、中国/跨境相关性、可建联性。
- repo：star 势能、维护状态、技术深度、商业化潜力、maintainer 可信度、主题匹配、可建联性。
- paper：技术新颖性、商业化路径、作者可信度、实现可用性、主题匹配、时效性、中国/跨境相关性。
- post：信号质量、互动质量、可追踪性、新鲜度、主题匹配、建联路径。

然后应用封顶和扣分：

- 没有 URL：raw score 最高 50。
- 没有明确邮箱：outreach score 最高 50。
- 帖子无法追踪到真实实体：raw score 最高 65。
- 纯 AI wrapper：扣 20。
- 明显太晚期：扣 15。

## 邮件模板

系统保留英文和中文两套模板：

- 英文：`Technical exchange around {project}`
- 中文：`想就 {project} 做一次技术交流`

邮件定位是技术交流、合作探讨和早期研究兴趣，不是一上来硬推投资。

## 定时任务建议

业务逻辑在 Python 中：

```bash
PYTHONPATH=src python3 -m sourcing_agent.cli run-daily
```

定时触发可以有两种方式：

- MVP 阶段：用 Codex automation 每天触发这个命令。
- 稳定运行阶段：迁移到 cron、GitHub Actions、Cloud Scheduler 等。

这样系统不会绑定在某一个调度平台上。
